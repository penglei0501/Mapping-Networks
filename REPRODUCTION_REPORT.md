## 1. 项目背景

深度神经网络通常直接优化卷积层、全连接层或循环层中的全部权重。随着模型规模增加，直接训练高维权重空间会带来显存占用、优化困难和过拟合等问题。

论文 **Mapping Networks** 提出一种低维重参数化方法：不直接把目标网络的全部权重作为可训练参数，而是训练一个低维潜变量 `z`，再通过固定映射生成目标网络参数 `theta`：

```text
低维潜变量 z -> 固定映射 G -> 目标参数 theta -> 目标网络前向传播
```

训练时，任务损失的梯度经过目标网络和固定映射回传到潜变量，优化器主要更新潜变量而不是目标网络的全部参数。论文采用两种训练方式：

- **Ours\*** / **SLVT**：Single Latent Vector Training，用一个潜变量生成整个目标网络的参数。
- **Ours†** / **LWT**：Layer-Wise Training，为不同层使用独立潜变量，再分别生成每层参数。

论文还提出包含任务、稳定性、平滑性和对齐项的 Mapping Loss，并研究了低秩分解、剪枝等附加压缩方法。

## 2. 复现目的

本项目的目标不是只追求一个最高准确率，而是系统验证以下问题：

1. 少量潜变量能否生成可用于分类和时间序列预测的完整网络参数。
2. SLVT 和 LWT 能否在显著减少可训练参数的同时接近普通网络性能。
3. Mapping Loss 的相关约束能否稳定潜变量到权重空间的映射。
4. Mapping Network 是否能与低秩分解和剪枝结合。
5. 论文表格中的参数量、指标和趋势是否可以在当前代码与硬件环境中复现。

## 4. Table 1：Image Classification with Mapping CNN

### 4.1 这项复现做了什么

Table 1 检验 Mapping Network 在基础图像分类任务上的有效性。CNN1 和 CNN2 是直接训练全部权重的基线；Ours\* 使用单个潜变量生成全部参数；Ours† 使用分层潜变量生成各层参数。该实验同时比较参数压缩程度与分类准确率，是整篇论文最基础的有效性验证。

### 4.2 论文与复现结果

### 4.3 结果解释

- 两个普通 CNN 基线复现良好，其中 CNN2 在 MNIST 和 FashionMNIST 上均超过论文值。
- Mapping CNN 在 MNIST 上总体接近论文，CNN2 的 SLVT-2048 只相差 0.58 pp，LWT-2688 相差 0.83 pp。
- FashionMNIST 明显更难，所有 Mapping 配置均低于论文，说明低维映射对数据复杂度、映射尺度和训练协议更加敏感。
- 增加潜变量通常能提高结果，但并非严格单调；初始化、分层分配和 Mapping Loss 系数都会影响最终性能。
- 因此 Table 1 可视为“机制与参数预算复现完成”，但部分 FashionMNIST 行属于部分数值复现。

## 5. Table 4：Mapping LSTM on Air Pollution Dataset

### 5.1 这项复现做了什么

Table 4 将 Mapping Network 从 CNN 扩展到 LSTM，用过去 24 小时的多站点污染和气象特征预测 Aotizhongxin 站下一小时 PM2.5。预处理后的输入含 67 个特征，按时间顺序使用前 80% 训练、后 20% 测试，归一化只在训练段拟合。

普通 LSTM 直接训练 12,961 个参数；Mapping LSTM 则用不同维度的潜变量生成 LSTM 参数。该实验验证方法是否适用于序列模型，以及潜变量维度与回归误差之间的关系。

### 5.2 论文与复现结果

| Method | # Params | 论文 MSE | 复现 MSE | 复现 - 论文 |
|---|---:|---:|---:|---:|
| LSTM | 12,961 | 0.00350 | **0.000298** | -0.003202 |
| Ours\* | 64 | 0.00190 | **0.001849** | -0.000051 |
| Ours\* | 256 | 0.00093 | 0.001613 | +0.000683 |
| Ours\* | 512 | 0.00080 | 0.001442 | +0.000642 |
| Ours\* | 1,024 | 0.00065 | 0.001330 | +0.000680 |
| Ours\* | 2,048 | 0.00061 | 0.000748 | +0.000138 |

其中 64 和 2,048 是正文 Table 4 的主要档位；256、512、1,024 来自补充材料的扩展参数档位。

### 5.3 结果解释

- 64 维 Mapping LSTM 与论文最接近，误差只相差约 `5.1e-5`。
- 潜变量从 256 增至 2,048 时，复现 MSE 总体下降，支持“潜变量容量增加可提高拟合能力”的趋势。
- 普通 LSTM 的 MSE 显著优于论文，说明当前数据来源、特征构造、归一化或时间切分与作者私有协议并不完全一致，不能仅凭更低 MSE 宣称严格超过论文。
- Table 4 的参数趋势得到复现，但数据协议差异是解释绝对数值时的重要限制。

## 6. Table 7：Robustness Study of Mapping CNN

### 6.1 这项复现做了什么

Table 7 通过改变 Mapping Network 的组成来判断性能究竟来自潜变量、固定映射还是权重调制：

- **Full DNN**：固定潜变量，训练较大的映射网络。
- **Ours\* - WM**：保留潜变量和固定正交映射，但关闭 weight modulation。
- **Ours\***：标准单潜变量 Mapping Network。
- **LV + WMAP**：潜变量和独立调制向量同时训练。
- **LV + FullDNN**：同时训练潜变量和无偏置稠密映射矩阵。

该表不是简单比较模型大小，而是研究各组成部分对准确率、稳定性和参数效率的贡献。

### 6.2 论文与复现结果

| Method | # Params | MNIST 论文 | MNIST 复现（Delta） | FMNIST 论文 | FMNIST 复现（Delta） |
|---|---:|---:|---:|---:|---:|
| CNN2 | 108,618 | 98.69% | 99.06% (+0.37) | 90.40% | 91.19% (+0.79) |
| Full DNN | 6,753,104 | 97.12% | **99.12% (+2.00)** | 90.11% | 90.10% (-0.01) |
| Ours\* - WM | 1,024 | 95.62% | 97.51% (+1.89) | 86.51% | 84.73% (-1.78) |
| Ours\* - WM | 2,048 | 96.55% | 98.29% (+1.74) | 87.66% | 86.43% (-1.23) |
| Ours\* | 1,024 | 97.88% | 97.10% (-0.78) | 89.49% | 85.04% (-4.45) |
| Ours\* | 2,048 | 98.66% | 98.08% (-0.58) | 91.88% | 86.82% (-5.06) |
| LV + WMAP | 2,048 | 97.90% | 97.33% (-0.57) | 89.30% | 84.52% (-4.78) |
| LV + WMAP | 4,096 | 98.48% | 98.18% (-0.30) | 91.93% | 86.22% (-5.71) |
| LV + FullDNN | 543,095 | 96.16% | **99.21% (+3.05)** | 90.11% | **90.66% (+0.55)** |
| LV + FullDNN | 1,629,285 | 97.60% | **99.16% (+1.56)** | 90.67% | 90.30% (-0.37) |

### 6.3 结果解释

- 无调制版本仍能取得较高准确率，说明固定投影加潜变量本身已具备较强表达能力。
- 增大无调制潜变量从 1,024 到 2,048 后，两套数据上的结果均提高。
- LV + FullDNN 的容量远高于标准 Mapping Network，复现结果超过论文并不代表其参数效率更好。
- Full DNN 的具体架构未在论文中披露；本项目使用满足论文参数预算的显式假设实现，因此该行属于“参数量一致的推断性复现”。
- MNIST 结果整体稳定，FashionMNIST 对调制方式和映射结构更敏感。

## 7. Table 8：Impact of Add-Ons on Mapping Network

### 7.1 这项复现做了什么

Table 8 检查低秩分解和剪枝能否与普通 CNN、SLVT、LWT 组合：

- **LRD**：将 CNN2 最大的全连接权重分解为两个低秩矩阵，降低实际生成或存储的目标参数量。
- **Prune**：按权重幅值保留 10% 目标参数，并在固定掩码下微调。

论文正文称 LRD rank 为 16，但 Table 8 的 35,914 参数只有在 rank=32 且仅分解 CNN2 的 FC1 时才能精确得到。本项目采用 rank=32，并把这一论文内部不一致记录在结果文件中。剪枝细节未充分披露，本项目采用 tensor-scope magnitude pruning，最终精确保留 10,862 个非零目标参数。

### 7.2 论文与复现结果

| Method | # Params | MNIST 论文 | MNIST 复现（Delta） | FMNIST 论文 | FMNIST 复现（Delta） |
|---|---:|---:|---:|---:|---:|
| CNN2 | 108,618 | 98.69% | 99.06% (+0.37) | 90.40% | 91.19% (+0.79) |
| CNN2 + LRD | 35,914 | 98.12% | **99.11% (+0.99)** | 89.67% | **90.43% (+0.76)** |
| CNN2 + Prune | 10,862 | 95.87% | **97.90% (+2.03)** | 87.91% | 86.44% (-1.47) |
| Ours\* | 2,048 | 98.66% | 98.08% (-0.58) | 91.88% | 86.82% (-5.06) |
| Ours\* + LRD | 1,456 | 97.80% | 97.02% (-0.78) | 90.67% | 85.27% (-5.40) |
| Ours\* + Prune | 2,048 | 95.93% | 94.92% (-1.01) | 88.70% | 80.65% (-8.05) |
| Ours† | 2,688 | 99.28% | 98.35% (-0.93) | 94.08% | 87.38% (-6.70) |
| Ours† + LRD | 2,688 | 98.81% | 98.34% (-0.47) | 93.55% | 85.71% (-7.84) |
| Ours† + Prune | 2,688 | 97.15% | 91.62% (-5.53) | 91.79% | 79.50% (-12.29) |

### 7.3 结果解释

- 直接训练的 CNN2 与 LRD、Prune 结合时复现效果最好，MNIST 上均超过论文。
- SLVT + LRD 在 MNIST 上仅低 0.78 pp，说明低秩目标参数可以由较小潜变量有效生成。
- Mapping + Prune 在 FashionMNIST 上差距最大。一次性剪除 90% 目标参数会使初始准确率接近随机值，固定掩码微调虽能恢复，但受到原始 dense Mapping 模型上限和短训练预算限制。
- LWT + Prune / FashionMNIST 最终采用累计微调后的 79.50%，该行是流程正确但数值差距明显的部分复现。
- 由于论文没有披露剪枝范围、调度和微调轮数，当前结果不能证明论文数值错误，只能说明本项目采用的明确剪枝协议没有复现其全部性能。

## 8. 总体结论

1. **核心机制得到验证。** 少量潜变量可以通过固定映射生成可工作的 CNN 和 LSTM 参数。
2. **参数压缩趋势得到验证。** 多个 Mapping 配置用远少于目标网络的可训练参数获得了接近基线的结果。
3. **MNIST 最容易复现。** Table 1、7、8 中多项 MNIST 结果与论文接近，部分基线和高容量消融超过论文。
4. **FashionMNIST 是主要差距来源。** 尤其是 LWT、LRD 和高稀疏率剪枝组合，对映射初始化、潜变量分配、损失和训练轮数非常敏感。
5. **Table 4 的趋势可信但协议不完全一致。** 64 维结果非常接近论文，但普通 LSTM 显著更好，提示数据预处理和切分存在差异。
6. **不能把“运行完成”写成“所有数值完全复现”。** 本项目已经完成 Table 1、4、7、8 的实现、参数核对和实验执行，其中同时包含接近复现、超过论文和部分复现三类结果。

## 9. 复现限制

- 论文没有完整披露所有实验的 epoch、batch size、学习率、优化器和数据归一化方式。
- 分层潜变量的具体分配和每层 modulation scale 未完整披露。
- Full DNN 架构、剪枝调度和 LRD rank 存在缺失或内部不一致。
- 当前主要报告单个随机种子 `42`，尚未统计多随机种子的均值和标准差。
- 实验跨 WSL 与服务器运行，Python、PyTorch 和 GPU 环境并非完全统一。
- Table 4 使用可审计的 UCI 数据预处理协议，但无法确认与作者未公开的数据处理完全相同。
